# Security Policy

We build tools that real people rely on, so we take security seriously — even
for small projects.

## Reporting a vulnerability

**Please do not open a public issue for security problems.**

Instead, report privately via GitHub's
[private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
on the affected repository (the **Security → Report a vulnerability** tab), or
contact a steward directly.

Please include:

- Which tool/repo is affected.
- What the vulnerability is and how to reproduce it.
- The potential impact, as best you understand it.

## What to expect

- We'll acknowledge your report as quickly as we reasonably can — we're
  volunteers, so please be patient.
- We'll work with you to understand and confirm the issue.
- We'll fix it in the open once users can safely update, and we'll credit you
  unless you prefer to stay anonymous.

Thank you for helping keep the commons safe.
